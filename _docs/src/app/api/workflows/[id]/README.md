## Overview

This directory is focused on server-side request/response handling for resources identified by a workflow ID. At the root it contains a single TypeScript route implementation (route.ts) that implements the primary route handler logic associated with operations scoped to an individual workflow ID. That root file is intended to encapsulate how incoming requests are handled and how responses are formulated for workflow-specific requests.

The preview subdirectory contains an additional route implementation (preview/route.ts) that specifically implements preview-related behavior for the same workflow ID. Together, the root route.ts and preview/route.ts provide the entry points for general workflow-id-scoped handlers and preview functionality respectively. The structure separates the main workflow route from preview-specific behavior so developers can find and modify preview handling without changing the primary route implementation.


### File Organization

Files are organized by feature/scope (routes tied to a workflow ID). The root-level route.ts provides the primary workflow-id route handler, while the preview subdirectory isolates preview-specific route logic. This keeps preview handling separate and discoverable while keeping the top-level route concise.
