## Overview

This directory is the reports area of the application's API surface and contains no root-level source files; instead, it is organized into subdirectories that each host a single route implementation file (route.ts) or further nested routing structure. Each subdirectory encapsulates a focused piece of the reports API: agents, errors, inbound, outbound, and overview each contain a route.ts intended to implement request/response handling for that specific report endpoint. The calls subdirectory currently contains no files at the root and is organized to provide per-identifier handling in a nested [id] route directory.

Together, these subdirectories provide the report-related endpoints for the application. Developers should treat each subdirectory as the canonical location for the implementation of a particular report route: review the respective route.ts to understand request handling, response shapes, and error handling behavior for that area. This directory functions as the reports namespace within the larger API, grouping all report routes and related error/inbound/outbound handling in one place to simplify navigation and maintenance.


### File Organization

No root-level files; functionality is separated into subdirectories by report type or role. Each populated subdirectory contains a single route.ts implementation (agents, errors, inbound, outbound, overview). The calls subdirectory is organized to host identifier-scoped routes under a nested [id] directory rather than root files.
