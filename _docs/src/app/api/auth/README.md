## Overview

This directory serves as the collection point for authentication API route implementations. At the root level there are no standalone files; instead the implementation is organized into subdirectories by endpoint. Each subdirectory contains a single TypeScript route.ts file that acts as the entry point for handling requests for that specific piece of the authentication flow.

The two subdirectories present are focused on discrete authentication responsibilities: login/ contains the implementation for the login API route and forgot-password/ contains the implementation for the forgot-password API route. Together these endpoint-specific directories implement pieces of the overall authentication functionality exposed by the API, with each route.ts acting as the entry point for incoming requests related to its endpoint.


### File Organization

Files are organized by feature/endpoint: each subdirectory under this auth directory corresponds to a single API endpoint and contains a route.ts file as the entry point for that endpoint. There are no files at the auth root; functionality is discovered by entering the endpoint folders.
