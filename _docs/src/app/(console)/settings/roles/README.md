That root page acts as the integration point where role-specific pages and UI are surfaced to users navigating the console settings.

Role-specific implementations are collected in subdirectories (for example the editor/ subdirectory). Each subdirectory contains the role-specific settings page implementation used by the console settings -> roles area, providing the UI entry point and related files for that role's settings. Together the root page.tsx and the role subdirectories form a small, hierarchical feature area: the root page composes and routes to specific role pages located in subfolders, while each role folder contains the UI and supporting files for that role's settings screen.


### File Organization

A single root-level entry point (page.tsx) provides the page component for the roles settings area. Role-specific implementations are placed in subdirectories (one per role) so each role's UI and related files are isolated and easy to maintain. This separation keeps the root page focused on composition/routing while subfolders contain role-specific logic and components.
