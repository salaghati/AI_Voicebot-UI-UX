## Overview

This directory contains the preview views used by the console route for a specific workflow identified by id. At the root level there are no files; instead the directory delegates each preview mode to a dedicated subdirectory. Each subdirectory provides a single route-level page component (page.tsx) that implements the UI for a specific preview experience: API logs, conversation, knowledge-base (KB), and session previews.

The subdirectories are organized so each preview mode is isolated: api-log/ provides the API log preview UI for a workflow, conversation/ renders the conversation preview, kb/ implements the KB preview, and session/ contains the session preview page. Together these components form the full set of preview entry points under the (console)/workflow/[id]/preview path and allow the larger console routing system to render the appropriate preview view for a given workflow id by delegating to the matching subdirectory page component.


### File Organization

There are no files at the preview directory root. Functionality is separated into one-page-per-mode subdirectories (api-log, conversation, kb, session). Each subdirectory contains a single route-level page component implementation (page.tsx) responsible for that preview mode, keeping concerns isolated and easy to locate.
