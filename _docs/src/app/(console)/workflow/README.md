## Overview

This directory provides the route-level implementation for workflow-related console pages. At the root it contains a single page.tsx file which serves as the main page component for the workflow route, acting as the entry point for viewing or interacting with workflows in the console. The root page.tsx is intended to render or coordinate workflow-level UI for the console area.

The directory also contains two route subdirectories: [id] and new. The [id] subdirectory holds the route-level page and any related route subviews for a specific workflow identified by the dynamic [id] route segment; its root page.tsx is the main route component for viewing or interacting with a particular workflow. The new subdirectory contains the route-level page.tsx that implements the console page used when creating or initializing a new workflow. Together, the root page.tsx and the route-specific pages in [id]/ and new/ provide the pages necessary to list, view, and create workflows within the console route hierarchy.


### File Organization

Files are organized by route-level responsibilities. The root directory contains a single page-level component (page.tsx) that represents the workflow route entry. Route-specific behaviors and views are placed under subdirectories named for their route segments ([id] for dynamic per-workflow views and new for the creation flow), each containing its own page.tsx. This keeps each route's implementation collocated with its path segment.
