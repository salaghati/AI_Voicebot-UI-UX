## Overview

The report directory groups the console report-related route implementations and page components. There are no files at the root of this directory; instead, functionality is split into focused subdirectories, each containing the page.tsx implementation for a single report route (for example: agent-analysis, error-monitor, inbound, outbound, overview). One subdirectory (call-detail) is a route segment that currently contains no files at its root but provides a dynamic subsegment [id] to render an individual call detail view.

Together these subdirectories form the console's reporting surface: each subdirectory represents a distinct route under docs/src/app/(console)/report and contains the TSX page responsible for rendering that route. The directory's role in the larger system is to centralize all console-facing report pages so routing and per-report UIs are colocated by route name. Because the root contains no files, developers will navigate directly into the listed subdirectories to find the page.tsx implementations for each report view or the dynamic call detail segment implementation.


### File Organization

There are no root-level files; the directory is organized into one directory per report route. Each report subdirectory contains a single page.tsx file implementing that route's page. The call-detail route is a route segment that defers implementation to a nested dynamic [id] subdirectory. This organization groups route-specific code by feature (route) to make it straightforward to locate and modify individual report pages.
