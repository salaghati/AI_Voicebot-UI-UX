## Overview

This directory holds the outbound section of the bot-engine console UI. At the root level there is a single page component (page.tsx) which acts as the primary route component for the outbound listing or overview within the bot-engine console area. That file represents the main entry point for the outbound console route at this path.

The directory is organized into three subdirectories that break out route-specific functionality: a dynamic [id]/ folder for page-level implementations keyed by a dynamic identifier (used to render or handle outbound bot-engine views for a particular id), a create/ folder that contains the TSX page implementing the outbound create view for the console, and a new/ folder that contains the step-specific page components for the multi-step 'new' outbound bot creation flow (organized into step-1 through step-4 subdirectories). Together these files and subdirectories provide the page-level routing and UI components required to view, create, and step through outbound bot-engine flows within the console.


### File Organization

Root-level route component(s) live at this directory's root (page.tsx). Route-specific functionality is split into subdirectories: [id]/ for dynamic id routes, create/ for the create view, and new/ for the multi-step creation flow. This groups pages by route/feature and keeps each page component in its own folder for clarity.
