## Overview

This directory is a route-focused container for the outbound bot creation flow's UI pages. There are no root-level files; instead, the directory organizes four step subdirectories (step-1 through step-4). Each subdirectory contains a single page component (page.tsx) that implements the UI and route-level logic for that particular step in the multi-step outbound bot creation process.

Collectively, the subdirectories represent the sequence of screens a user moves through when creating a new outbound bot in the console. The pages in step-1, step-2, step-3, and step-4 are intended to be composed by the surrounding application routing and navigation. This structure keeps each step isolated to its own folder so developers can reason about and modify a single step without touching other steps, while the larger bot-engine outbound new flow is composed by the platform-level routing and navigation logic outside this directory.


### File Organization

There are no root files; the directory is organized into per-step subdirectories (step-1 ... step-4). Each subdirectory contains a single page-level component named page.tsx representing that step's UI. This keeps step implementations isolated and easy to navigate.
