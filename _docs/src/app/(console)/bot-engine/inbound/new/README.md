## Overview

This directory (docs/src/app/(console)/bot-engine/inbound/new) serves as a container for the route-level page implementations that make up the step-by-step inbound 'new' flow in the bot-engine console area. There are no root-level source files in this directory; instead, each step of the flow is implemented as a single entry file (page.tsx) inside its own subdirectory. Each subdirectory (step-1, step-2, step-3, step-4) contains the route component that represents the UI endpoint for that particular step.

Collectively, the subdirectories define the multi-step user interface for creating or configuring inbound bot-engine items in the console documentation application. The structure groups each step's page implementation into its own folder so the route entry point and any step-specific assets or code (if present) are colocated. This directory functions as the logical container for the inbound/new flow's pages and provides the single location developers should inspect when modifying or reviewing the route-level pages for each step.


### File Organization

There are no files at the root of this directory. Each step of the inbound/new flow is implemented in its own subdirectory (step-1 through step-4), and each subdirectory contains a single route-level entry file named page.tsx. This keeps each step isolated and makes it clear where to find the route component for a given step.
