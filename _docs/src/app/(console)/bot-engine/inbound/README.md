## Overview

This directory contains the route-level implementation artifacts for the inbound portion of the bot-engine console. At the root level there is a single page component file (page.tsx) that represents the primary route implementation for this directory. That root page file serves as the entry point for the inbound route segment defined at this location.

The directory is organized into three subdirectories: [id], create, and new. The [id] subdirectory contains the dynamic segment implementation for individual inbound bot-engine resources and holds a single page.tsx component. The create subdirectory contains a single page.tsx that implements the inbound create route. The new subdirectory is a container for a step-by-step inbound "new" flow and does not contain root-level files itself; it is organized into per-step subdirectories (not listed here). Together, the root page.tsx and the route-level pages in the subdirectories form the complete inbound routing surface for this portion of the console, with each page component acting as the route entry point for its respective segment.


### File Organization

Files are organized by route segment: a single root-level page.tsx implements the inbound route at this level, while subdirectories group route implementations for dynamic segments ([id]), a create route (create), and a multi-step new flow (new). Each route-level folder contains a page.tsx as the canonical entry point for that route segment.
