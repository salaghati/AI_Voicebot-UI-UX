## Overview

This directory holds the route-level page that serves as the campaigns index under the console documentation site and a nested subdirectory that contains the multi-step new campaign flow. At the root level there is a single route file (page.tsx) which represents the campaigns landing page for this section of the documentation site. The root page file provides the entry point for users navigating to the bot-engine campaigns area.

The new/ subdirectory is the container for the multi-step "new campaign" flow. It does not contain root-level files itself but organizes the creation flow into four step subdirectories (step-1 through step-4), each of which includes a route-level TypeScript React page component (as described in the subdirectory overview). Together, the root page.tsx and the new/ flow implement the campaigns area: page.tsx acts as the index/entry, and the new/ subdirectory implements the guided, step-based creation screens that users progress through when creating a campaign.


### File Organization

The directory uses a hierarchical, route-oriented organization: a single root-level route page (page.tsx) serves as the campaigns index and a nested new/ directory contains the multi-step campaign creation flow. The nested structure groups related route components for the multi-step flow under new/ for clarity and routing consistency.
