## Overview

This directory serves as the consolidated location for console preview UI routes and pages related to the preview experience. At the root there are no standalone files; instead, the work is organized into focused subdirectories. Each subdirectory provides a single page-level entry for a specific preview area: a platform-review area with a root page and further route organization for inbound, outbound, knowledge-base, and workflow preview flows, and a playground area that contains a page to render and exercise the console preview UI for this route.

Together, the subdirectories in this directory act as route-level entry points for the console preview functionality: platform-review provides the entry page and structured route implementations for multiple review-related preview flows, while playground provides an isolated page to exercise and visually test the preview UI. This organization centralizes preview route implementations so the console preview surface is discoverable and separated by feature area, making it easier to navigate and maintain the preview pages within the larger app routing structure.


### File Organization

There are no files at the directory root; functionality is split into feature-focused subdirectories. Each subdirectory contains a single page-level entry (page.tsx) that implements the preview UI or route entry for that feature area. This keeps route entry points grouped by feature (platform-review, playground) and isolates their implementations.
