## Overview

This directory contains the route-level page implementation files used to render preview and create views for inbound platform reviews inside the preview console area. At the root it exposes a single page implementation (page.tsx) intended to be the entry point for the inbound platform review preview route. That root page.tsx represents the primary page component invoked for this route and is the main artifact present at this directory level.

Two subdirectories provide route-specific pages: the [id] subdirectory contains the route-level page component used to render the inbound platform review preview for a specific dynamic id segment, and the create subdirectory contains the page component that renders the create screen for inbound platform review within the preview console flow. Together, the root-level page.tsx and the page.tsx files inside the two subdirectories form a small, hierarchical set of route components that implement preview, per-id preview, and create screens for inbound platform reviews. This structure locates route implementations together to make it straightforward to find and modify UI for the inbound preview console flows.


### File Organization

A small hierarchical route layout: a single root-level page.tsx acts as the directory's primary page, with two child subdirectories ([id] and create) each containing their own route-level page.tsx to handle dynamic-id previews and create screens respectively. Files are organized by route responsibility to keep related page implementations adjacent.
