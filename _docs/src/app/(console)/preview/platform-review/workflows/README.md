## Overview

This directory contains the route-level implementation for previewing platform-review workflow instances within the console preview UI. At the root level there are no direct files; instead the functionality is organized under a dynamic route subdirectory named [id]. That subdirectory contains a single TypeScript React page (page.tsx) which is the actual rendering and entry point for an individual workflow preview. The root of this directory therefore serves as the container and namespace for per-workflow preview routes rather than holding implementation files itself.

The [id] subdirectory represents a dynamic route segment under console/preview/platform-review/workflows and holds the page.tsx that implements the preview UI for a specific workflow identified by the dynamic segment. Together, the directory and its [id] child provide a hierarchical routing structure: the workflows directory groups all per-workflow preview routes, and each [id] folder holds the code required to render a single workflow's preview. This structure makes it clear where to add or locate the UI for a specific workflow preview instance and how it integrates into the larger console preview system.


### File Organization

There are no files at the workflows root; implementation is organized by dynamic route subdirectories. Each dynamic segment ([id]) contains a single page.tsx that acts as the entry point for rendering a workflow preview. This keeps route-level code colocated with its dynamic identifier and makes it straightforward to locate per-workflow page logic.
