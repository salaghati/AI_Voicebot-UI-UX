## Overview

This directory provides the entry point and route-level implementations for the platform-review preview area of the console. At the root there is a single page implementation (page.tsx) which serves as the primary entry or component for the platform-review preview route. That root file is intended to bootstrap or render the preview UI for platform review scenarios within the preview console area.

The directory is organized into focused subdirectories: inbound and outbound contain concrete route-level page components and preview screens for their respective flows (each exposing one or more route implementations). The kb subdirectory is reserved for knowledge-base preview pages and is structured under a dynamic route, while workflows exposes per-workflow preview routes using a dynamic segment. Together these pieces form the platform-review preview surface: the root page.tsx acts as the entry, and each subdirectory provides specialized route implementations for different facets of platform-review previews (inbound, outbound, KB items, and workflow instances).


### File Organization

Files are organized by feature/route area: a single root entry file (page.tsx) exists at this directory root, while feature-specific route implementations and screens are grouped into subdirectories (inbound, outbound, kb, workflows). This groups route-level UI by flow and keeps preview implementations localized.
