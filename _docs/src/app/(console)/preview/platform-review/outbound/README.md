## Overview

This directory contains the root-level outbound preview page implementation and two focused subdirectories for specific outbound preview routes. At the root there is a single implemented file, page.tsx, which serves as the component or entry file for the outbound platform-review preview. That root-level page.tsx is expected to represent or bootstrap the outbound preview for platform-review requests at this path.

Two subdirectories provide route-specialized implementations: the create/ subdirectory contains a page.tsx implementing the outbound "create" preview, and the [id]/ subdirectory holds a page.tsx that implements the outbound preview for a specific platform-review identified by the dynamic [id] segment. Together, the root page.tsx and these subdirectories form a small hierarchical route structure: the root file is the primary entry for outbound previews, while the subdirectories contain route-specific pages that the router can load for create or per-id views.


### File Organization

The directory is organized as a small route-focused hierarchy: a single root entry file (page.tsx) at the directory root with two route subdirectories (create and [id]) each containing their own page.tsx. This keeps route-specific implementations isolated in their subfolders while maintaining a single, discoverable root entry point.
