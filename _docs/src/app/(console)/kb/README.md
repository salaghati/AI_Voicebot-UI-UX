## Overview

The kb directory is the console knowledge-base area of the documentation application. At the root it contains a single route-level page file (page.tsx) that serves as the canonical entry for this directory and is expected to implement or coordinate the primary UI for the /console/kb route. That root page.tsx is the primary entry point for this directory and ties together navigation and high-level rendering responsibilities for knowledge-base related views.

Beneath the root are five focused subdirectories (add, fallback, list, usage, version). Each subdirectory contains a single route-level page.tsx that implements the specific sub-route UI: 'add' for creating KB items, 'fallback' for rendering and managing fallback item views and related actions, 'list' for rendering list views and handling list-item routes, 'usage' for usage-related views and per-id/topic usage pages, and 'version' for the version route. Together these route files form a hierarchical feature set where the root page.tsx provides the top-level entry and each subdirectory page.tsx implements its route-specific UI and behavior, forming the console/kb routing surface of the docs site.


### File Organization

Files are organized by route responsibility: a single root-level page.tsx serves as the directory entry, and focused subdirectories each contain their own page.tsx implementing route-level UI for distinct KB concerns (add, fallback, list, usage, version). This keeps route implementations localized and predictable.
