## Overview

This directory contains the route-level TSX page component(s) and subfolders that implement the console knowledge-base "usage" views. At the root there is a page.tsx file which is the primary route-level module for the /console/kb/usage route. That file serves as the entry point for rendering or handling the usage view tied to the usage listing or overview for the console knowledge-base section. It is intended to be the code entry for presenting usage-related content and wiring route behavior at this path.

Two subdirectories provide targeted usage pages: the [id] folder contains a single route-level TSX page component for rendering a usage view specific to a particular id param (i.e., the per-resource usage page), and the filter folder contains a page.tsx that renders the documentation page focused on the 'filter' topic under the console knowledge-base usage section. Together, the root page.tsx and these subdirectory page components form a small hierarchical set of route handlers and documentation pages: the root-level file provides the main usage view entry, while the subdirectories provide more specific or parameterized pages that the routing layer will resolve under /console/kb/usage/[id] and /console/kb/usage/filter respectively. This structure organizes general usage UI at the root and topic-/id-specific pages in dedicated folders so developers can locate and modify route-level components for each usage view.


### File Organization

Root-level page.tsx acts as the primary route entry for /console/kb/usage. Parameterized and topic-specific routes are placed in dedicated subdirectories ([id] and filter), each containing a single page.tsx to represent that route. This keeps route components colocated with their URL structure and makes it straightforward to add new topic or parameterized usage pages by adding new subfolders with page.tsx modules.
