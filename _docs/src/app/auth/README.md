## Overview

This directory holds the authentication route implementations for the application. There are no files at the directory root; instead, each auth route is implemented as a separate subdirectory that contains a single TypeScript JSX page component (page.tsx). The root-level organization is deliberately minimal: each route's visual and presentation logic lives in its own folder to keep page responsibilities isolated.

The forgot-password/ subdirectory contains one TypeScript React page component (page.tsx) that is responsible for the forgot-password UI and related presentation logic for that route. The login/ subdirectory contains one TypeScript React page component (page.tsx) that serves as the top-level page file for the auth/login route. Together, these subdirectories provide the route entry points for authentication flows; each page.tsx is the primary implementation artifact for its route and is intended to be the page-level component that the routing layer will render.


### File Organization

There are no root-level files; the directory is organized into one subdirectory per auth route. Each subdirectory contains a single page.tsx file that acts as the route's page-level component and presentation entry point. This keeps each route self-contained and makes it straightforward to add or modify auth pages by working within their respective folders.
