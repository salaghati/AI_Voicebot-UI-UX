## Overview

This directory houses four TSX component files that together represent the visual building blocks for the workflow feature. Each file corresponds to a distinct UI responsibility implied by its filename: a builder interface, a detailed view, a diagram/canvas renderer, and a preview view. These components are intended to be used wherever workflow creation, inspection, diagramming, or previewing is required within the workflow feature.

At the root level the files provide specialized responsibilities: WorkflowBuilder.tsx for the workflow construction interface, WorkflowDetailView.tsx for showing detailed information about a workflow, WorkflowDiagramCanvas.tsx for rendering the workflow diagram or canvas, and WorkflowPreviewView.tsx for rendering a preview of a workflow. Together they form a cohesive set of components that a developer can assemble when implementing or extending the workflow UI. There are no subdirectories listed for this directory.


### File Organization

Files are organized by feature responsibility: each TSX file represents a specific UI component for the workflow feature (builder, detail view, diagram canvas, and preview). This keeps related UI components colocated for easier maintenance and discovery.
