## Overview

This directory contains the primary UI components and a metadata module for the kb (knowledge-base) feature. The root-level files include view components (KbDetailView.tsx, KbFallbackDetailView.tsx), editor and form components for creating and editing entries (KbEditForm.tsx, KbFallbackEditor.tsx, KbSourceForm.tsx), a shell/wrapper component (KbShell.tsx) and a TypeScript metadata file (kb-fallback-meta.ts). Collectively these files provide the visual and interaction surface required to display, edit, and handle fallback scenarios for knowledge-base content.

Files are organized so that presentation and editing concerns are colocated: detail views and fallback detail views focus on read/display functionality; edit and source forms handle authoring and source-related input; the shell component provides a common wrapper or orchestration entrypoint for composing these pieces in pages or routes; and the kb-fallback-meta.ts file contains metadata used by fallback components. Together they form the component-level building blocks that the larger kb feature imports and composes into higher-level pages or features in the application.


### File Organization

Files are grouped by role within the kb feature: views, editors/forms, fallback variants, and a metadata module. This keeps related UI components together for easier discovery and composition when building kb pages.
