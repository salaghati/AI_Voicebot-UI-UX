## Overview

The _docs directory contains configuration files at the root and two focused subdirectories that together support the documentation application and its end-to-end validation. At the root there are three TypeScript configuration files (next.config.ts, playwright.config.ts, vitest.config.ts). These files serve as the visible configuration entry points found alongside the docs source and testing specifications and are intended to configure runtime/build/test behavior for the documentation application and its test suites.

Below the root, the e2e/ subdirectory contains three TypeScript end-to-end specification files that validate full flows and visual/voice behaviors; these spec files are the executable test scenarios for validating integrated behaviors. The src/ subdirectory is the documentation application source root and is organized into focused subdirectories (route components, UI components, feature entry modules, shared libraries, tests, and TypeScript domain types) rather than top-level files. Together, the root config files, the e2e specifications, and the src application sources form a cohesive documentation workspace where configuration controls build/test behavior, src contains the application implementation, and e2e contains validation scenarios that exercise the application end-to-end.


### File Organization

Root-level TypeScript configuration files are placed at the top of the directory as primary entry points for build and test tooling, while implementation and test specifications are separated into subdirectories: src/ for the documentation application source and e2e/ for end-to-end specification files. This keeps configuration discoverable at the directory root and groups source and tests into clear folders.
